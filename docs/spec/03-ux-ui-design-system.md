# UI/UX

# **FILOSOFIA DE UX/UI E DESIGN SYSTEM**

## **Objetivo**

A interface deverá proporcionar uma experiência moderna, intuitiva e agradável, priorizando clareza, produtividade e conforto visual durante longos períodos de utilização.

O usuário deverá sentir que está utilizando um sistema leve, organizado e inteligente, onde as informações fluem naturalmente e o foco permanece no conteúdo, não na interface.

O design deverá transmitir sofisticação e simplicidade ao mesmo tempo, evitando tanto a aparência rígida de sistemas corporativos tradicionais quanto o excesso de efeitos visuais encontrados em interfaces extremamente decorativas.

---

# **Conceito Visual**

O sistema deverá seguir uma identidade visual inspirada em interfaces modernas de produtividade.

As principais referências são aplicativos utilizados em dispositivos móveis, adaptados para desktop.

Características desejadas:

* sensação de profundidade;  
* componentes levemente elevados;  
* cantos arredondados;  
* transparência discreta;  
* sombras suaves;  
* degradês muito sutis;  
* organização limpa;  
* bastante espaço em branco;  
* elementos espelhados e foscos;  
* animações discretas;  
* interface elegante;  
* baixo contraste

O sistema não deverá utilizar aparência "quadrada" ou excessivamente corporativa.

---

# **Estilo Visual**

O sistema deverá transmitir as seguintes sensações:

* moderno;  
* leve;  
* organizado;  
* sofisticado;  
* acolhedor;  
* elegante;  
* minimalista;  
* profissional.

Evitar aparência de:

* ERP;  
* sistema financeiro;  
* software antigo;  
* planilha;  
* Windows Forms;  
* interfaces extremamente técnicas.

# **Fundo do Sistema**

O fundo principal deverá utilizar uma tonalidade clara e quente.

```
#EEF4ED
```

Objetivos:

* reduzir fadiga visual;  
* transmitir conforto;  
* valorizar os componentes.

O fundo nunca deverá competir visualmente com os cards.

---

# **Tipografia**

A tipografia deverá priorizar leitura.

Características:

* moderna;  
* limpa;  
* excelente legibilidade;  
* boa diferenciação entre títulos e conteúdo.

Hierarquia:

Título Principal

↓

Título Secundário

↓

Subtítulo

↓

Texto

↓

Informações auxiliares

A hierarquia deverá ser percebida pelo tamanho, peso e espaçamento, não apenas pela cor.

# **Cards**

Os cards representam o principal elemento visual do sistema.

Características:

* cantos arredondados;  
* sombra extremamente suave;  
* fundo sólido;  
* leve sensação de profundidade;  
* espaçamento interno confortável.

Evitar:

* bordas pesadas;  
* sombras fortes;  
* excesso de transparência;  
* muitos contornos.

---

**Menu lateral**

deverá ser igual em todas as telas e deverá ocupar o mínimo de espaço possível

---

# **Transparência**

A transparência deverá existir apenas para criar sensação de leveza.

Nunca deverá comprometer a leitura.

Não utilizar efeito "vidro" exagerado.

---

# **Animações**

Todas as animações deverão ser rápidas.

Objetivos:

* indicar interação;  
* melhorar percepção;  
* aumentar sensação de qualidade.

Nunca utilizar animações longas.

Evitar efeitos chamativos.

---

# **Layout**

O layout deverá evitar sensação de "caixas empilhadas".

As informações deverão parecer distribuídas naturalmente.

Características desejadas:

* bastante respiro;  
* alinhamentos consistentes;  
* poucos divisores;  
* agrupamentos naturais;  
* sensação de continuidade;  
* nenhum elemento, campo, card deve ser extremamente grande

---

# **Componentes**

Todos os componentes deverão seguir o mesmo padrão visual.

Exemplos:

Botões

Campos

Filtros

Dropdowns

Calendário

Cards

Gráficos

Tabelas

Modais

Menus

Badges

Indicadores

Todos deverão parecer parte do mesmo sistema.

---

# **Estados**

Todo componente deverá possuir estados visuais consistentes.

Exemplos:

Normal

Hover

Selecionado

Focado

Desabilitado

Erro

Sucesso

Carregando

Todos deverão seguir o mesmo padrão visual.

---

# **Responsividade**

O sistema deverá ser Desktop First.

Posteriormente adaptar-se para:

Tablet

Celular

A adaptação nunca poderá esconder funcionalidades importantes.

No celular deverão ser reorganizadas.

Nunca removidas.

---

# **Regras Obrigatórias**

Durante toda evolução do sistema deverão ser respeitadas as seguintes regras:

* Não alterar a paleta oficial.  
* Não criar novos estilos para componentes já existentes.  
* Não misturar estilos diferentes.  
* Não utilizar mais de uma linguagem visual.  
* Manter consistência em todas as telas.  
* Toda nova tela deverá seguir exatamente o mesmo padrão definido neste documento.  
* Ajustar o texto de todos os campos quando ficar grande:  Quando um texto for inserido em um campo e esse texto tiver número de caracteres maior que o número de caracteres que são exibidos dentro do campo, o texto e o campo devem ser ajustados, para que tudo aquilo que foi registrado no campo apareça automaticamente, sem que haja a ocultação de caracteres. para todos os campos de texto disponíveis, você deverá habilitar a possibilidade do campo ser ajustado nesse sentido, quando o texto exceder a quantidade de caracteres. 

---

# **Experiência Esperada**

Ao abrir o sistema, o usuário deverá perceber imediatamente:

* organização;  
* leveza;  
* rapidez;  
* clareza;  
* modernidade;  
* conforto visual.

A interface deverá transmitir confiança e facilitar a concentração no trabalho, reduzindo distrações e destacando apenas as informações realmente importantes.

sempre que uma nova tela for criada, priorizar reutilizar componentes, padrões e comportamentos já existentes, em vez de inventar um layout diferente.
